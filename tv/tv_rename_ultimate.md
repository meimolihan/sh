# 电视剧文件重命名脚本使用说明

```bash
bash <(curl -sL gitee.com/meimolihan/sh/raw/master/tv/tv_rename_ultimate.sh)
```



## 📁 脚本概述

`tv_rename_ultimate.sh`是一个智能化的电视剧文件重命名工具，专门设计用于批量重命名电视剧视频文件，使其符合刮削器（如Plex、Emby、Jellyfin等）的命名规范。

### 🎯 主要功能特性

- 

  **智能多模式识别** - 支持6种不同的剧集编号格式

- 

  **自动检测方案** - 智能分析文件名并推荐最佳重命名方案

- 

  **双模式操作** - 快速模式（一键操作）和专业模式（详细设置）

- 

  **实时预览** - 重命名前可预览所有更改

- 

  **安全保护** - 不会直接修改文件，需用户确认后才执行

- 

  **详细统计** - 提供识别准确率和文件统计信息

## 📂 支持的视频格式

脚本支持以下视频文件扩展名：

- 

  `.mp4``.mkv``.avi``.mov``.wmv`

- 

  `.flv``.webm``.m4v``.mpg``.mpeg``.kvm`

## 🔍 支持的识别模式

### 6种智能识别模式

1. 

   **SxE格式** - 标准剧集编号

   - 

     示例：`S01E26`, `S1E1`

   - 

     匹配：`成何体统.How.Dare.You.S01E26.2026.2160p.IQ.WEB-DL.H265.DDP5.1-ColorWEB.mkv`

2. 

   **EP格式** - 简写剧集编号

   - 

     示例：`EP26`, `EP26-`, `Episode1`

   - 

     匹配：`EP26-剧情发展.mp4`

3. 

   **中文第X集格式** - 中文剧集标识

   - 

     示例：`第26集`, `第1集`

   - 

     匹配：`第26集.电视剧名.avi`

4. 

   **数字格式** - 中间数字

   - 

     示例：`26-`, `.26.`

   - 

     匹配：`test-26-video.mkv`

5. 

   **开头数字** - 文件名开头的数字

   - 

     示例：`01.文件名`, `1-剧集名`

   - 

     匹配：`01.第一集.mkv`

6. 

   **结尾数字** - 扩展名前的数字

   - 

     示例：`文件名.26`, `剧集-26`

   - 

     匹配：`video.26.mp4`

## 🛠️ 详细使用指南

### 准备工作

1. 

   **文件组织**：

   - 

     将所有要重命名的电视剧文件放在同一目录

   - 

     建议一个目录只放一季的剧集

   - 

     确保文件名中包含剧集编号信息

2. 

   **备份重要文件**（可选但推荐）：

   - 

     在执行重命名前备份原始文件

   - 

     脚本不会删除文件，但会修改文件名

### 基本使用步骤

#### 步骤1：进入目标目录

```
cd /path/to/your/tv-series/
```

#### 步骤2：运行脚本

```
./tv_rename_ultimate.sh
```

#### 步骤3：选择重命名模式

- 

  新手用户：选择 **1. 快速重命名**

- 

  高级用户：选择 **2. 详细重命名**

#### 步骤4：设置重命名参数

**快速模式**会自动设置以下参数：

- 

  剧集前缀：从文件名智能提取

- 

  季号：从SxE格式自动识别

- 

  起始集数：从识别的最小集数开始

**详细模式**可手动设置：

- 

  剧集前缀：剧集的主要名称

  - 

    示例：输入"成何体统" → 生成`成何体统-S01E26.mkv`

- 

  季号：剧集的季数

  - 

    示例：输入"01" → 显示为S01

  - 

    支持1-99季

- 

  起始集数：从第几集开始编号

  - 

    示例：输入"26" → 从E26开始编号

  - 

    支持1-999集

#### 步骤5：预览重命名结果

- 

  脚本会显示每个文件的改名预览

- 

  显示原始文件名和识别模式

- 

  显示新的标准化文件名

- 

  提供识别准确率统计

#### 步骤6：确认执行

- 

  输入 `y`确认执行重命名

- 

  输入 `N`取消操作

- 

  脚本会显示重命名进度和结果

### 高级功能使用

- 

  **选项a**：提取中文剧名部分

#### 季号自动识别

#### 集数识别优化

-

- 

  支持多种数字格式识别

- 

  按识别的集数排序文件

## 🔧 故障排除

### 常见问题

#### 1. 脚本无法识别文件

- 

  **原因**：文件名不包含标准剧集编号

- 

  **解决**：手动设置前缀和起始集数

- 

  **建议**：确保文件名包含`S01E26`、`EP26`或`第26集`等格式

#### 2. 识别准确率低

- 

  **原因**：文件名格式不统一

- 

  **解决**：使用详细模式手动调整

- 

  **建议**：批量重命名前先统一文件名格式

#### 3. 季号识别错误

- 

  **原因**：文件名包含多个数字

- 

  **解决**：手动设置正确的季号

- 

  **建议**：检查文件名中是否有年份等干扰数字

#### 4. 权限问题

- 

  **解决**：确保对目录有写权限

- 

  命令：`chmod +w /path/to/directory`
