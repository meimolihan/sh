# Shell脚本中256色终端颜色的使用指南

```bash
bash <(curl -sL gitee.com/meimolihan/sh/raw/master/color/256color.sh)
```



在开发命令行工具或编写Shell脚本时，为输出添加颜色可以极大提升用户体验和可读性。现代终端普遍支持256色，但如何快速查找并正确使用这些颜色代码呢？本文将介绍一个实用的脚本——`256color_escape.sh`，它能生成完整的256色表，并附带每个颜色的转义码，帮助你轻松在脚本中应用颜色。

## 1. 脚本简介

`256color.sh` 是一个Bash脚本，用于在终端中显示0-255所有256色的背景色块，并在每个色块旁边以黑色背景打印出对应的**前景色转义码**（例如 `\033[38;5;21m`）。它的主要特点包括：

- 自动根据背景色亮度选择黑色或白色文字，保证色块上的数字清晰可读。
- 将颜色分为三组展示：基本色（0-15）、216色立方体（16-231）和灰度色（232-255）。
- 每个颜色单元包含：背景色块（显示颜色代码） + 空格 + 前景色转义码文本（背景黑）。
- 最后提供常用颜色示例和特殊效果演示。

通过这个脚本，你可以直观地看到每一种颜色的实际效果，并直接复制对应的转义码用于自己的脚本中。

## 2. 脚本内容与输出格式

### 2.1 核心函数

脚本中定义了几个关键函数：

- `choose_fg()`：根据背景色的亮度（基于RGB估算）返回合适的前景色代码（0黑色或15白色），确保色块上的数字可读。
- `print_color_cell()`：打印单个颜色单元，格式为：  
  `[背景色块(显示代码)] [空格] [前景色转义码文本]`

### 2.2 输出结构

运行脚本后，终端将依次显示：

1. **基本颜色 (0-15)**：每行3个单元，包含经典的系统颜色及亮色。
2. **216色 (16-231)**：6×6×6的RGB立方体颜色，同样每行3个单元。
3. **灰度色 (232-255)**：24级灰度，每行3个单元。
4. **常用颜色示例**：每行一个，包含颜色单元和简短描述（如“21:蓝”）。
5. **使用说明**：包括前景色/背景色的转义序列格式，以及特殊效果示例。
6. **额外示例**：包含你提供的示例（如15-25颜色预览）。

### 2.3 输出示例

```
=== 256色终端颜色预览 (带转义码) ===
每个单元：背景色块 | 空格 | 前景色转义码文本（背景黑）
基本色/216色/灰度色：每行显示3个单元

=== 基本颜色 (0-15) ===
    0 \033[38;5;0m      1 \033[38;5;1m      2 \033[38;5;2m
    3 \033[38;5;3m      4 \033[38;5;4m      5 \033[38;5;5m
...
```

## 3. 如何使用这个脚本

### 3.1 获取脚本

将脚本内容保存为文件，例如 `256color.sh`，并赋予执行权限：

```bash
chmod +x 256color.sh
```

### 3.2 运行脚本

直接在终端中运行：

```bash
./256color.sh
```

如果希望将输出保存到文件以供日后参考，可以重定向：

```bash
./256color.sh > color_table.txt
```

但注意，重定向会丢失颜色效果，仅保留文本。建议在终端中直接查看。

### 3.3 解读输出

- **背景色块**：显示颜色代码（如 `21`），背景色即为该代码对应的颜色。
- **转义码**：如 `\033[38;5;21m`，表示将后续文字前景色设为颜色21（蓝色）。你可以直接复制这段文本到你的脚本中使用。

例如，输出中有一行：
```
   21 \033[38;5;21m
```
这意味着颜色21是蓝色，其前景色转义码为 `\033[38;5;21m`。

## 4. 在Shell脚本中集成颜色代码

### 4.1 基本用法

在Shell脚本中，使用`echo -e`或`printf`来解析转义序列。例如：

```bash
#!/bin/bash
echo -e "\033[38;5;21m这是蓝色文字\033[0m"
```

- `\033[38;5;21m`：设置前景色为颜色21。
- `\033[0m`：重置所有属性，恢复默认颜色。

背景色使用 `\033[48;5;Nm`，例如：

```bash
echo -e "\033[48;5;226m黄色背景\033[0m"
```

### 4.2 组合使用

可以同时设置前景色、背景色和文字样式：

```bash
echo -e "\033[1;38;5;196;48;5;226m粗体红字黄背景\033[0m"
```

其中 `1` 表示粗体，多个属性用分号分隔。

### 4.3 定义颜色变量

为了使脚本更易维护，可以定义颜色变量：

```bash
#!/bin/bash
BLUE='\033[38;5;21m'
YELLOW_BG='\033[48;5;226m'
RESET='\033[0m'

echo -e "${BLUE}这是蓝色文字${RESET}"
echo -e "${YELLOW_BG}黄色背景${RESET}"
```

### 4.4 在脚本中动态选择颜色

你可以根据条件选择不同的颜色代码。例如，根据日志级别设置颜色：

```bash
log() {
    local level=$1
    local msg=$2
    case $level in
        INFO)  color='\033[38;5;82m';;  # 亮绿
        WARN)  color='\033[38;5;214m';; # 橙色
        ERROR) color='\033[38;5;196m';; # 亮红
        *)     color='\033[0m';;
    esac
    echo -e "${color}[$level] $msg${RESET}"
}
```

## 5. 注意事项

- **终端兼容性**：并非所有终端都支持256色。大多数现代终端（如GNOME Terminal、iTerm2、Konsole、Windows Terminal等）都支持。可以通过 `echo $TERM` 查看终端类型，通常包含 `256color` 字样（如 `xterm-256color`）。
- **转义序列格式**：`\033` 是ESC字符的八进制表示，也可用 `\e` 或 `\x1b` 代替。在`echo`中需要`-e`选项，或使用`printf`。
- **重置属性**：始终在颜色输出后使用 `\033[0m` 重置，以免影响后续输出。
- **背景色与前景色对比度**：在深色背景上使用深色文字会导致难以阅读。脚本中的 `choose_fg` 函数提供了参考，但在自己的脚本中，你可能需要根据背景手动选择合适的文字颜色。

## 6. 扩展应用

- **自定义提示符**：可以在PS1中使用颜色代码，例如：
  ```bash
  export PS1='\[\033[38;5;39m\]\u@\h\[\033[0m\]:\[\033[38;5;226m\]\w\[\033[0m\]\$ '
  ```
- **生成配色方案**：将脚本输出导入文本编辑器，筛选喜欢的颜色代码，构建自己的配色库。
- **调试输出**：为不同级别的调试信息分配不同颜色，快速定位问题。

## 7. 总结

`256color_escape.sh` 是一个简单但强大的工具，它将抽象的256色代码转化为可视化的参考，并直接提供可复用的转义码。通过在Shell脚本中合理运用这些颜色，你可以让命令行输出更加生动、直观。希望本文能帮助你掌握256色终端颜色的使用方法，提升脚本的交互体验。
